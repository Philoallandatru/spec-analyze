# PCIe Retimer（重定时器）

Retimer 是 PCIe Gen 3 及以上引入的物理层设备，用于延长链路距离、改善信号质量。它通过接收、重定时并重新发送信号，解决高速串行链路中的信号完整性问题，使 PCIe 能够支持更长的走线和更复杂的拓扑结构。

---

## 概述

### 什么是 Retimer？

**Retimer（重定时器）** 是一个物理层透明设备，位于 PCIe 链路中间，执行以下操作：
- **接收衰减信号**：从上游接收已衰减的高速串行信号
- **时钟数据恢复 (CDR)**：提取时钟并恢复数据
- **重定时**：使用恢复的时钟对数据重新采样
- **重新发送**：以全幅度、低抖动重新发送信号到下游

**关键特性**：
- ✅ **物理层透明**：对事务层和数据链路层完全不可见
- ✅ **协议无感知**：不解析 TLP/DLLP，仅处理编码符号
- ✅ **零延迟增加**（几乎）：仅增加几纳秒传播延迟
- ✅ **支持热插拔**：可在链路训练时动态检测
- ✅ **Gen 3+ 必需**：8 GT/s 及以上速率的长距离链路

### 为什么需要 Retimer？

**信号完整性挑战**：

| PCIe Gen | 信号速率 | 主要挑战 | 典型走线长度 |
|----------|---------|---------|------------|
| Gen 1-2 | 2.5-5 GT/s | 适中 | 20-50 cm |
| Gen 3 | 8 GT/s | 高 | 10-15 cm (无 Retimer) |
| Gen 4 | 16 GT/s | 极高 | 7-10 cm (无 Retimer) |
| Gen 5 | 32 GT/s | 严峻 | <7 cm (无 Retimer) |

**Gen 3+ 的问题**：
```
发送端                     接收端
  │                         │
  │  ▁▂▃▄▅▆▇█ (全幅度)       │
  │                         │
  │  走线损耗 + 反射 + ISI    │
  │                         │
  │          ▁▂▃▂▁ (衰减)   │  ← 眼图闭合
  │                         │
  ▼                         ▼
无法可靠恢复数据

解决方案：插入 Retimer
发送端 ──→ Retimer ──→ 接收端
  8 cm        │      8 cm
  (可行)      │      (可行)
              ↓
         重新生成全幅度信号
```

### Retimer vs Redriver

**对比**：

| 特性 | Redriver（中继器） | Retimer（重定时器） |
|------|------------------|-------------------|
| **工作层次** | 模拟信号放大 | 数字重定时 + 再生 |
| **CDR** | 无 | 有（完整时钟恢复） |
| **抖动清除** | 部分（均衡） | 完全（重新生成） |
| **功耗** | 低（~100 mW） | 较高（~500 mW） |
| **延迟** | <1 ns | 2-5 ns |
| **适用场景** | Gen 1-2 短距离 | Gen 3+ 长距离 |
| **成本** | 低 | 中等 |
| **协议依赖** | 无 | 需支持编码方案 |

**选择建议**：
- **Redriver**：Gen 1-2，走线 <20 cm，成本敏感
- **Retimer**：Gen 3+，走线 >15 cm，信号质量关键

## Retimer 架构

### 基本结构

```
       ┌─────────────────────────────────────┐
       │          Retimer 芯片               │
       │                                     │
上游端口│  ┌──────────┐      ┌──────────┐   │下游端口
───────┼─→│ RX Path  │      │ TX Path  │───┼───────→
Lane 0 │  │          │      │          │   │ Lane 0
       │  │ ┌──────┐ │      │ ┌──────┐ │   │
       │  │ │ CDR  │ │      │ │ Ser  │ │   │
       │  │ │      │ │      │ │      │ │   │
       │  │ │解串化│ │FIFO  │ │串行化│ │   │
       │  │ │8b/10b│─┼─────→│128b/  │ │   │
       │  │ │解码  │ │      │130b   │ │   │
       │  │ │      │ │      │编码   │ │   │
       │  │ └──────┘ │      │ └──────┘ │   │
       │  └──────────┘      └──────────┘   │
       │       ↓                  ↑         │
       │  ┌────────────────────────────┐   │
       │  │   链路训练状态机 (LTSSM)   │   │
       │  │   - 检测 Ordered Sets      │   │
       │  │   - SKP 插入/删除          │   │
       │  │   - 伪重定时标记           │   │
       │  └────────────────────────────┘   │
       │                                     │
       │  时钟：独立 PLL × 2（上游+下游）   │
       └─────────────────────────────────────┘
```

### 数据流

**接收路径 (RX)**：
```
1. 差分信号输入 → 模拟前端 (AFE)
2. 时钟数据恢复 (CDR) → 提取时钟
3. 解串化 (Deserializer) → 并行数据
4. 解码 (8b/10b 或 128b/130b) → 原始数据
5. FIFO 缓冲 → 时钟域切换
```

**发送路径 (TX)**：
```
1. FIFO 读取 → 本地时钟域
2. 编码 (8b/10b 或 128b/130b) → 编码数据
3. Scrambling（Gen 3+） → 扰码
4. 串行化 (Serializer) → 串行比特流
5. 输出驱动器 → 差分信号输出
```

### FIFO 和时钟管理

**时钟域隔离**：
```
上游时钟域              下游时钟域
     │                      │
     │   ┌──────────┐       │
RX──→├──→│ WR FIFO  │──→──→├──→ TX
     │   │          │       │
     │   │  深度:   │       │
     │   │  16-32   │       │
     │   │  符号    │       │
     │   └──────────┘       │
     │                      │
 独立 PLL            独立 PLL
```

**SKP 字符处理**：
- **FIFO 快满**：删除 SKP（上游时钟慢）
- **FIFO 快空**：插入 SKP（上游时钟快）
- 每 1180 符号检查一次

## 链路训练中的 Retimer

### 伪重定时标记（Pseudo-Compliance Pattern）

Retimer 在链路训练期间需要被检测，通过修改训练序列实现：

**TS1 训练序列（无 Retimer）**：
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│COM │TS1 │Link│Lane│...│TS1 │TS1 │END │
│K28.5│ ID │Num│Num │   │ ID │ ID │... │
└────┴────┴────┴────┴────┴────┴────┴────┘
    16 个符号序列，每个 TS1 = 16 符号
```

**TS1 通过 Retimer（添加标记）**：
```
Retimer 修改 Symbol 4（Link Number）：
  原始: Link Number (正常值)
  修改: 设置 Bit 0 = 1 (Pseudo 标记)

接收端检测到 Bit 0 = 1 → 知道链路包含 Retimer
```

**目的**：
- 允许接收端调整均衡参数
- 告知系统链路延迟增加
- 启用 Retimer 特定的错误处理

### 链路训练流程

```
Root Complex (RC)         Retimer              Endpoint (EP)
     │                       │                      │
     ├──TS1 (速度协商)──────→│                      │
     │                       ├──TS1 (添加标记)─────→│
     │                       │                      │
     │                       │←─TS1 (响应)─────────┤
     │←─TS1 (转发)───────────┤                      │
     │                       │                      │
     ├──TS2 (确认)──────────→│                      │
     │                       ├──TS2 (转发)─────────→│
     │                       │                      │
     │                       │←─TS2 (确认)─────────┤
     │←─TS2 (转发)───────────┤                      │
     │                       │                      │
     ├──数据包 (L0 状态)────→│                      │
     │                       ├──重定时并转发───────→│
     │                       │                      │
```

### Retimer 数量检测

**多个 Retimer 场景**：
```
RC ──[Retimer 1]── [Retimer 2] ── EP

每个 Retimer 在 TS1 中设置标记
接收端通过分析训练序列模式检测 Retimer 数量
规范允许最多 2 个 Retimer
```

## 实现和配置

### 硬件实现

**典型 Retimer 芯片**：
- **Astera Labs**: Aries PCIe/CXL Retimer
- **Microchip**: PM8534/PM8546 (PCIe Gen 4/5)
- **Parade Technologies**: PS9405 (PCIe Gen 5)

**关键参数**：
```
- 通道数: x1, x2, x4, x8, x16
- 支持速率: Gen 3 (8 GT/s), Gen 4 (16 GT/s), Gen 5 (32 GT/s)
- CDR 带宽: 50-100 MHz
- 抖动生成: <0.3 UI RMS
- 功耗: 300-800 mW（取决于通道数和速率）
- 封装: BGA, QFN
```


### 软件可见性

**重要**：Retimer 对软件**完全透明**，但可通过以下方式间接检测：

**方法 1：链路延迟测量**：
```bash
# 使用精确时间戳测量往返延迟
# 含 Retimer 的链路延迟约增加 5-10 ns
```

**方法 2：链路状态寄存器**（部分平台）：

## 性能影响

### 延迟

**延迟组成**：
```
总延迟 = 传播延迟 + Retimer 延迟

无 Retimer（15 cm 走线）:
  传播延迟: 15 cm ÷ 15 cm/ns = 1 ns
  总延迟: ~1 ns

有 Retimer（2 段 × 8 cm）:
  传播延迟: 16 cm ÷ 15 cm/ns = 1.07 ns
  Retimer 延迟: 3-5 ns (CDR + FIFO + 串行化)
  总延迟: ~5 ns

对比: 增加 ~4 ns
```

**影响评估**：
- **存储 I/O**：单次 4KB 读取 ~10 μs，4 ns 增加可忽略（0.04%）
- **低延迟网络**：可能影响 <1 μs 延迟目标，需要评估
- **GPU 互连**：PCIe 5.0 x16 带宽 64 GB/s，延迟增加影响小

### 带宽

**Retimer 不影响理论带宽**：
```
Gen 4 x16 (无 Retimer): 31.5 GB/s
Gen 4 x16 (有 Retimer): 31.5 GB/s

原因: Retimer 工作在物理层，不改变编码或数据率
```

**实际吞吐量**：
```bash
# 测试实际带宽（fio + NVMe）
fio --name=seq_read --rw=read --bs=128k --numjobs=4 \
    --iodepth=32 --filename=/dev/nvme0n1 --direct=1

无 Retimer: ~6.8 GB/s (Gen 4 x4 NVMe)
有 Retimer: ~6.8 GB/s (差异 <1%)
```

### 信号质量

**眼图对比**：
```
无 Retimer（20 cm 走线，Gen 4）:
  眼高: 40 mV (规范要求 60 mV) ❌
  眼宽: 0.5 UI (规范要求 0.6 UI) ❌
  抖动: 0.35 UI (规范要求 <0.3 UI) ❌
  结果: 链路训练失败

有 Retimer（2 × 10 cm，Gen 4）:
  眼高: 120 mV ✅
  眼宽: 0.7 UI ✅
  抖动: 0.2 UI ✅
  结果: 链路稳定运行
```

## 调试和故障排除

### 常见问题

**问题 1：链路无法训练到目标速度**
```
症状: lspci 显示 Speed 5GT/s (downgraded)
      期望 16GT/s (Gen 4)

可能原因:
1. Retimer 固件版本不兼容
2. Retimer 时钟配置错误
3. 走线质量问题（即使有 Retimer）

调试步骤:
# 检查链路状态
lspci -vvv -s 01:00.0 | grep -i "lnkcap\|lnksta"

# 查看 AER 错误
setpci -s 01:00.0 ECAP_AER+0x04.L  # UESta
setpci -s 01:00.0 ECAP_AER+0x10.L  # CESta

# 尝试降速测试
setpci -s 01:00.0 CAP_EXP+30.w=0032  # Target Gen 3
setpci -s 01:00.0 CAP_EXP+10.w=0020  # Retrain
```

**问题 2：间歇性数据错误**
```
症状: PCIe AER 报告 Correctable Errors
      dmesg 显示 "Receiver Error"

可能原因:
1. FIFO 溢出（SKP 处理失败）
2. 时钟抖动过大
3. 温度漂移

解决方法:
# 检查 SKP 处理
# Retimer 应每 ~1180 符号处理 SKP
# 如果 FIFO 深度不足，可能丢失数据

# 监控错误率
watch -n 1 'lspci -vvv -s 01:00.0 | grep -A2 "CESta"'
```

**问题 3：热插拔失败**
```
症状: 热插拔设备后链路无法建立

可能原因:
1. Retimer 未正确复位
2. 电源时序问题
3. PERST# 信号异常

调试:
# 查看热插拔事件
dmesg | grep -i "pcie.*hotplug"

# 手动触发链路重训
echo 1 > /sys/bus/pci/devices/0000:01:00.0/remove
echo 1 > /sys/bus/pci/rescan
```

## 高级主题

### Gen 5 (32 GT/s) 挑战

**更严格的要求**：
```
Gen 4 (16 GT/s)          Gen 5 (32 GT/s)
  │                        │
  ├ 抖动: <0.3 UI         ├ 抖动: <0.2 UI
  ├ 眼高: >60 mV          ├ 眼高: >40 mV (更小)
  ├ 走线: 7-10 cm         ├ 走线: <7 cm
  ├ Retimer: 可选         ├ Retimer: 几乎必需
  └ 均衡: CTLE + DFE      └ 均衡: 先进 FFE/DFE
```

**Gen 5 Retimer 特性**：
- **自适应均衡**：动态调整 FFE/DFE 参数
- **前向纠错 (FEC)**：某些芯片支持可选 FEC
- **链路裕量测试**：内置 BER 测试功能

### CXL (Compute Express Link) Retimer

**CXL 特殊要求**：
```
CXL 基于 PCIe 物理层，但增加了:
  - CXL.io (等同 PCIe)
  - CXL.cache (缓存一致性)
  - CXL.mem (内存语义)

Retimer 需求:
  ✅ 支持 68B FLIT 模式（CXL 特有）
  ✅ 更严格的延迟要求（<10 ns）
  ✅ 低抖动（缓存一致性敏感）
```

### 多 Retimer 级联

**规范限制**：
```
最多 2 个 Retimer:
RC ─[Retimer 1]─ [Retimer 2] ─ EP  ✅

3 个或更多:
RC ─[R1]─ [R2] ─ [R3] ─ EP  ❌ (不保证)

原因:
  - 累积延迟: 3 × 5 ns = 15 ns (影响重传计时)
  - 累积抖动: 可能超出规范
  - 训练复杂度: 伪标记检测困难
```

## 实用建议

### 设计指南

**何时使用 Retimer**：
1. ✅ 走线长度 >15 cm (Gen 3), >10 cm (Gen 4), >7 cm (Gen 5)
2. ✅ 多层板穿孔（Via）超过 2 个
3. ✅ 需要经过连接器（增加插入损耗）
4. ✅ 信号仿真显示眼图不符合规范

**不需要 Retimer**：
1. ❌ Gen 1-2 短距离链路（<30 cm）
2. ❌ 板载直连（CPU 到 SSD，<10 cm）
3. ❌ 成本极度敏感的消费类产品

### 采购考量

**关键参数清单**：
- [ ] 支持的 PCIe Gen（3/4/5）
- [ ] 通道数（x1/x2/x4/x8/x16）
- [ ] 功耗预算（含散热方案）
- [ ] 固件可升级性
- [ ] 温度范围（工业级 vs 商业级）
- [ ] 供应商支持和认证

**成本估算**：
```
Gen 3 x4 Retimer: $5-10
Gen 4 x4 Retimer: $10-20
Gen 5 x16 Retimer: $50-100

PCB 额外成本: $2-5（布线复杂度增加）
```

## 总结

### 关键要点

1. ✅ **Retimer 是物理层设备**，通过 CDR 重定时信号，对上层协议透明
2. ✅ **Gen 3+ 长距离链路必需**，解决高速信号完整性问题
3. ✅ **增加 3-5 ns 延迟**，但对大多数应用影响可忽略
4. ✅ **不改变带宽**，理论和实际吞吐量保持不变
5. ✅ **通过伪重定时标记**在链路训练时被检测
6. ✅ **规范允许最多 2 个 Retimer** 级联

**Retimer vs Redriver 速查**：

| 场景 | 推荐方案 |
|------|---------|
| Gen 3, 走线 20 cm | Retimer |
| Gen 4, 走线 15 cm | Retimer（必需）|
| Gen 5, 任何走线 >7 cm | Retimer（必需）|
| Gen 1-2, 走线 30 cm | Redriver 或无 |
| 成本敏感，短距离 | 无中继设备 |
