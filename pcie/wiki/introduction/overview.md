# PCIe 简介

## 什么是 PCI Express？

**PCI Express (PCIe)** 是一种高速串行计算机扩展总线标准，用于连接主板和外围设备。它是传统 PCI (Peripheral Component Interconnect) 总线的继任者。

### 核心特点

- **高速串行通信**：相比 PCI 的并行总线，PCIe 使用串行点对点连接
- **可扩展带宽**：支持多条通道（Lane）并行传输，从 x1 到 x32
- **向下兼容**：软件层面与 PCI 兼容，驱动迁移容易
- **低延迟**：点对点连接减少仲裁延迟
- **灵活拓扑**：支持复杂的树形拓扑结构

---

## 历史与演进

### 发展时间线

| 版本 | 发布年份 | 速度 (每通道) | 编码 | 有效带宽 (x16) |
|------|---------|--------------|------|---------------|
| **PCIe 1.0** | 2003 | 2.5 GT/s | 8b/10b | ~4 GB/s |
| **PCIe 2.0** | 2007 | 5.0 GT/s | 8b/10b | ~8 GB/s |
| **PCIe 3.0** | 2010 | 8.0 GT/s | 128b/130b | ~16 GB/s |
| **PCIe 4.0** | 2017 | 16.0 GT/s | 128b/130b | ~32 GB/s |
| **PCIe 5.0** | 2019 | 32.0 GT/s | 128b/130b | ~64 GB/s |
| **PCIe 6.0** | 2022 | 64.0 GT/s | PAM4 | ~128 GB/s |

**GT/s** = Giga Transfers per second (千兆传输/秒)

### 编码效率

- **8b/10b 编码**：每 8 位数据需要 10 位传输，效率 80%
  - 用于 PCIe 1.0 和 2.0
  
- **128b/130b 编码**：每 128 位数据需要 130 位传输，效率 98.5%
  - 用于 PCIe 3.0、4.0、5.0
  - 显著提高了带宽利用率

## 为什么选择 PCIe？

### 相比传统 PCI 的优势

#### 1. **更高的带宽**
```
PCI 32-bit/33MHz:  133 MB/s
PCI 64-bit/66MHz:  533 MB/s
PCIe 1.0 x16:      4 GB/s
PCIe 5.0 x16:      64 GB/s  ← 120倍提升！
```

#### 2. **点对点连接**
- **PCI**：共享总线，所有设备竞争带宽
- **PCIe**：每个设备有专用连接，无竞争

#### 3. **可扩展性**
- 可以根据需求配置通道数量（x1, x4, x8, x16）
- 低速设备用 x1，高速设备用 x16

#### 4. **串行差分信号**
- 更好的信号完整性
- 更少的引脚数量
- 更低的电磁干扰 (EMI)

#### 5. **热插拔支持**
- 原生支持热插拔（Hot-Plug）
- 无需重启即可添加/移除设备

## PCIe 的应用场景

### 常见 PCIe 设备

| 设备类型 | 典型通道数 | 带宽需求 |
|---------|----------|---------|
| **显卡 (GPU)** | x16 | 非常高 |
| **NVMe SSD** | x4 | 高 |
| **网卡 (10GbE+)** | x4/x8 | 中到高 |
| **RAID 控制器** | x8 | 高 |
| **声卡** | x1 | 低 |
| **USB 扩展卡** | x1 | 低 |
| **Wi-Fi 网卡** | x1 | 低 |

### 典型系统配置

```
┌────────────────────────────────────────────────────┐
│                    CPU                              │
│  (内置 PCIe Root Complex)                          │
└───┬─────────┬─────────┬──────────┬────────────────┘
    │         │         │          │
    │ x16     │ x4      │ x4       │ x1
    │         │         │          │
┌───▼───┐ ┌──▼───┐ ┌───▼────┐ ┌──▼────────┐
│  GPU  │ │ NVMe │ │ NVMe   │ │  其他设备 │
│       │ │ SSD  │ │  SSD   │ │           │
└───────┘ └──────┘ └────────┘ └───────────┘
```

## PCIe 基本术语

| 术语 | 含义 |
|------|------|
| **Lane (通道)** | 一对差分发送线 + 一对差分接收线，最小的数据传输单元 |
| **Link (链路)** | 由 1-32 条 Lane 组成的连接 |
| **Endpoint (端点)** | PCIe 设备，如显卡、网卡、SSD |
| **Root Complex (根复合体)** | CPU 内置的 PCIe 主控制器 |
| **Switch (交换机)** | 扩展 PCIe 连接的设备 |
| **TLP** | Transaction Layer Packet，事务层数据包 |
| **GT/s** | Giga Transfers per second，千兆传输/秒 |

## PCIe 架构预览

PCIe 采用**三层协议栈**架构：

```
┌─────────────────────────────────────────┐
│      Transaction Layer (事务层)         │
│  负责：TLP 生成、路由、排序、流控制      │
├─────────────────────────────────────────┤
│      Data Link Layer (数据链路层)       │
│  负责：可靠传输、ACK/NAK、重试、CRC     │
├─────────────────────────────────────────┤
│      Physical Layer (物理层)            │
│  负责：编码、链路训练、电气信号         │
└─────────────────────────────────────────┘
```

> 详细了解请参阅 [三层协议模型](../architecture/layering.md)

## PCIe 与其他总线对比

### PCIe vs SATA

| 特性 | PCIe (NVMe) | SATA |
|------|-------------|------|
| **接口** | 串行点对点 | 串行点对点 |
| **协议** | NVMe (优化为存储) | AHCI (为 HDD 设计) |
| **队列深度** | 65535 | 32 |
| **带宽 (Gen3)** | 4 GB/s (x4) | 600 MB/s |
| **延迟** | 更低 | 更高 |

### PCIe vs USB

| 特性 | PCIe | USB |
|------|------|-----|
| **目标** | 内部高速设备 | 外部通用设备 |
| **带宽** | 极高 (64 GB/s+) | 中等 (USB 3.2: 2.5 GB/s) |
| **距离** | 主板内/短距离 | 较长电缆 (3-5m) |
| **拓扑** | 树形，点对点 | 星形/树形，共享带宽 |
| **电源** | 通常需要独立供电 | 可总线供电 |

### PCIe vs Thunderbolt

**Thunderbolt** 实际上是 PCIe 和 DisplayPort 的外部封装：
- **Thunderbolt 3/4**：通过 USB-C 连接器传输 PCIe x4 信号
- 允许外接 PCIe 设备（eGPU、外置 RAID 等）
- 提供与内部 PCIe 相当的性能
