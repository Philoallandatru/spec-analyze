# PCIe 时钟架构

## 概述

PCIe 的时钟架构是系统设计中的关键要素，直接影响链路的稳定性、性能和互操作性。PCIe 采用**差分时钟**传输，支持多种时钟架构模式，并对时钟精度、抖动和频谱扩展有严格要求。

```
┌─────────────────────────────────────────────────────────┐
│            PCIe 时钟架构层次                             │
├─────────────────────────────────────────────────────────┤
│  参考时钟源 (100 MHz)                                   │
│    ├─> 共享时钟 (Common Clock)                         │
│    └─> 独立时钟 (Separate Reference Clock)            │
├─────────────────────────────────────────────────────────┤
│  PLL/时钟恢复电路                                       │
│    ├─> 时钟倍频                                        │
│    ├─> 相位锁定                                        │
│    └─> 抖动滤波                                        │
├─────────────────────────────────────────────────────────┤
│  串行器/解串器 (SerDes)                                 │
│    ├─> 数据采样时钟                                    │
│    └─> 发送时钟                                        │
└─────────────────────────────────────────────────────────┘
```

---

## 1. 参考时钟 (Reference Clock)

### 1.1 基本特性

PCIe 使用 **100 MHz** 差分参考时钟：

| 参数 | 规格 |
|------|------|
| 频率 | 100 MHz ± 300 ppm |
| 信号类型 | 差分 HCSL (High-speed Current Steering Logic) |
| 电压摆幅 | 250-700 mV (差分峰峰值) |
| 共模电压 | 250-650 mV |
| 占空比 | 40%-60% |

### 1.2 时钟信号命名

- **REFCLK+** / **REFCLK-**：差分时钟对
- 在连接器上通常为专用引脚
- Add-in Card 通常从主板接收参考时钟

---

## 2. 时钟架构模式

### 2.1 Common Clock (共享时钟)

**定义**：链路两端设备使用同一个物理时钟源。

```
┌──────────────┐
│ 100 MHz      │
│ Clock Source │
└───┬──────┬───┘
    │      │
    ▼      ▼
┌────────┐  ┌────────┐
│Root    │  │Endpoint│
│Complex │  │        │
└────────┘  └────────┘
```

**优势**：
- 时钟相位关系固定
- 更低的抖动累积
- 更简单的时钟恢复电路
- 更短的链路训练时间

**应用场景**：
- 主板上的板载设备
- 短距离互连
- 高性能应用

### 2.2 Separate Reference Clock (独立时钟)

**定义**：链路两端设备使用各自独立的时钟源。

```
┌──────────┐        ┌──────────┐
│Clock     │        │Clock     │
│Source A  │        │Source B  │
└────┬─────┘        └────┬─────┘
     │                   │
     ▼                   ▼
┌────────┐          ┌────────┐
│Root    │ ◄─────► │Endpoint│
│Complex │          │        │
└────────┘          └────────┘
```

**特点**：
- 时钟源频率可能存在差异（±300 ppm 范围内）
- 需要更复杂的时钟补偿机制（Clock Compensation）
- 支持独立电源域设计

**应用场景**：
- Add-in Card
- 跨系统互连
- 热插拔设备

### 2.3 Data Clock (数据时钟恢复)

无论使用哪种参考时钟模式，接收端都必须从接收的串行数据流中**恢复时钟**：

1. **CDR (Clock and Data Recovery)**：
   - 从编码数据中提取时钟信息
   - 8b/10b 或 128b/130b 编码保证足够的跳变
   - PLL 锁定到数据速率

2. **弹性缓冲区 (Elastic Buffer)**：
   - 补偿时钟频率差异
   - 插入/删除 SKP Ordered Sets 进行时钟补偿

## 3. 时钟精度要求

### 3.1 频率容差

| 时钟类型 | 容差 |
|----------|------|
| 参考时钟 (100 MHz) | ± 300 ppm |
| 串行比特率 | 从参考时钟派生 |

**计算示例**（Gen3 8.0 GT/s）：
- 标称频率：8.0 GHz
- 最大偏差：8.0 GHz × 300 ppm = ±2.4 MHz
- 频率范围：7.9976 ~ 8.0024 GHz

### 3.2 时钟抖动 (Jitter)

抖动是时钟信号边沿位置的时域偏差，分为：

**1. Random Jitter (随机抖动 - RJ)**
- 服从高斯分布
- 由热噪声、shot noise 等引起
- 无界，以 RMS 值表征

**2. Deterministic Jitter (确定性抖动 - DJ)**
- 可重复、可预测
- 包括周期性抖动、占空比失真
- 有界，以峰峰值表征

**3. Total Jitter (总抖动 - TJ)**
```
TJ = DJ + RJ(n-sigma)
```

### 3.3 抖动规格

不同代的 PCIe 有不同的抖动预算：

| Gen | 速率 | UI (Unit Interval) | 最大 TJ |
|-----|------|---------------------|---------|
| Gen1 | 2.5 GT/s | 400 ps | < 0.3 UI |
| Gen2 | 5.0 GT/s | 200 ps | < 0.3 UI |
| Gen3 | 8.0 GT/s | 125 ps | < 0.3 UI |
| Gen4 | 16.0 GT/s | 62.5 ps | < 0.3 UI |
| Gen5 | 32.0 GT/s | 31.25 ps | < 0.25 UI |

## 4. Spread Spectrum Clocking (SSC)

### 4.1 基本原理

**展频时钟**通过周期性地调制时钟频率，将电磁干扰 (EMI) 能量分散到更宽的频谱范围，降低峰值辐射。

```
频率
  │   非展频                 展频
  │     ▲                    ╱╲╱╲╱╲
  │     │                  ╱        ╲
  │     │                ╱            ╲
  ├─────┼───────────────────────────────► 时间
  │   f_center          f_center
  │
```

### 4.2 PCIe SSC 规格

| 参数 | 规格 |
|------|------|
| 调制类型 | 下展频 (Down-spreading) |
| 调制深度 | -0% ~ -0.5% |
| 调制频率 | 30-33 kHz |
| 调制波形 | 三角波 |

**为什么使用下展频？**
- 避免超过标称频率上限
- 保证时钟频率始终在规范范围内
- 简化接收端 CDR 电路设计

### 4.3 SSC 配置

通过 **Link Capabilities Register** 和 **Link Control Register** 协商：

- **SRIS (Separate Reference Clock with Independent SSC)**：
  - 允许链路两端使用独立的 SSC 配置
  - 需要更强的时钟补偿能力

## 5. 时钟补偿机制

### 5.1 SKP Ordered Set

在 **Separate Reference Clock** 模式下，由于两端时钟频率差异，需要周期性插入 **SKP (Skip) Ordered Sets** 进行补偿：

```
正常数据流：
[TLP] [TLP] [TLP] [TLP] ...

插入 SKP：
[TLP] [SKP] [TLP] [TLP] [SKP] [TLP] ...
```

**SKP 插入规则**：
- 每 **1180-1538 Symbol Times** 插入一次（Gen1/Gen2）
- Gen3+ 使用 128b/130b 编码，规则有所不同
- 接收端可以根据需要增删 SKP 符号

### 5.2 弹性缓冲区 (Elastic Buffer)

接收端使用弹性缓冲区吸收时钟频率差异：

```
发送时钟域 → [Elastic Buffer] → 接收时钟域
                  ↕
              SKP 插入/删除
```

**工作原理**：
1. 数据写入使用恢复的时钟
2. 数据读出使用本地时钟
3. 缓冲区深度动态调整
4. 通过 SKP 增删保持缓冲区在工作范围

## 6. 链路训练中的时钟锁定

### 6.1 LTSSM 中的时钟相关状态

```
Detect → Polling → Configuration → L0
   │        │            │          │
   ▼        ▼            ▼          ▼
检测接收端 锁定比特率  协商时钟模式  数据传输
```

**Polling 状态**：
- 接收端 CDR 锁定到发送端的比特率
- 交换 TS1/TS2 Ordered Sets
- 建立符号锁定和块对齐

**Configuration 状态**：
- 通过 Link Capabilities 交换时钟配置
- 确定 Common Clock 或 Separate Clock 模式
- 协商 SSC 配置

### 6.2 Common Clock 配置位

**Link Status Register (Offset 12h)** 中的 **Slot Clock Configuration** 位：
- **0**：设备使用独立时钟
- **1**：设备使用平台提供的时钟（Common Clock）

软件可以读取此位确定链路的时钟配置。

## 7. 时钟架构设计考虑

### 7.1 Common Clock 设计

**PCB 设计要点**：
```
Clock Source
     │
     ├─> RC (短走线，匹配长度)
     └─> EP (短走线，匹配长度)
```

- 时钟走线长度匹配（± 5 mils）
- 时钟与数据走线长度相关性
- 差分对阻抗控制（100Ω）
- 最小化时钟走线 stub

### 7.2 Separate Clock 设计

**隔离要求**：
- 各设备可以使用独立的晶振
- 支持异步电源域
- 需要保证 ±300 ppm 频率容差

### 7.3 时钟电源完整性

```
Clock Source
     │
     ▼
 [LDO/线性稳压器]
     │
     ▼
 [去耦电容网络]
     │
     ▼
Reference Clock Input
```

**关键点**：
- 使用低噪声电源
- 充足的去耦电容
- 独立的电源平面
- 避免数字噪声耦合

---

## 8. 时钟相关的电源管理

### 8.1 时钟门控

在 **L1 (低功耗状态)** 时：
- 可以关闭部分时钟树
- 保持必要的唤醒逻辑
- 参考时钟可能保持开启

### 8.2 时钟停止

在 **L1 子状态（L1.1, L1.2）**：
- 可以停止参考时钟（L1.2）
- 需要更长的恢复时间
- 通过 CLKREQ# 信号协调

**CLKREQ# 信号**：
- 由设备驱动
- 请求平台提供/停止参考时钟
- 支持动态时钟门控

## 9. 调试和测量

### 9.1 时钟测量点

在设计中应预留时钟测量点：
- 参考时钟输入
- PLL 输出
- 串行数据眼图观测

### 9.2 常见时钟问题

| 问题 | 症状 | 可能原因 |
|------|------|----------|
| 链路训练失败 | 停在 Polling | CDR 无法锁定，时钟质量差 |
| 间歇性错误 | CRC 错误 | 时钟抖动过大 |
| 带宽下降 | 频繁重训 | 时钟频率超出容差 |
| EMI 问题 | 辐射超标 | SSC 未启用或配置错误 |

### 9.3 测试工具

- **示波器**：测量时钟波形、抖动
- **频谱分析仪**：验证 SSC 展频效果
- **协议分析仪**：观察 SKP Ordered Sets
- **眼图测试**：评估整体信号质量

---

## 10. 不同代的时钟差异

### 10.1 Gen1/Gen2 (8b/10b 编码)

- 更宽松的抖动预算
- 简单的 CDR 实现
- 频繁的 SKP 插入（约每 1000 符号）

### 10.2 Gen3/Gen4 (128b/130b 编码)

- 更严格的抖动要求
- 更复杂的均衡和时钟恢复
- 更少的开销用于时钟补偿

### 10.3 Gen5+ (128b/130b + PAM4)

- 极严格的抖动预算（< 0.25 UI）
- 需要先进的 CDR 和均衡技术
- 对时钟源质量要求极高

---

## 总结

PCIe 时钟架构的关键要点：

1. **灵活性**：支持 Common Clock 和 Separate Clock 模式
2. **精度**：±300 ppm 频率容差，严格的抖动预算
3. **补偿**：通过 SKP Ordered Sets 和弹性缓冲区处理时钟差异
4. **EMI**：使用下展频 SSC 降低电磁辐射
5. **功耗**：支持动态时钟门控和 CLKREQ# 管理

正确的时钟架构设计是实现稳定、高性能 PCIe 链路的基础。
