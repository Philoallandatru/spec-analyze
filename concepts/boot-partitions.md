# Boot Partitions 启动分区

## 概述

启动分区（Boot Partitions）是 NVMe 控制器（Controller）提供的一项可选功能，用于存储系统启动相关的关键数据。它由**两个大小完全相同的分区**组成，形成一个分区对，为固件更新和启动代码提供了冗余保护机制。

### 核心特性

- **双分区设计**：包含分区 0 和分区 1，大小完全相同
- **主机控制读取**：主机（Host）选择分区、指定读取范围，并提供目标缓冲区
- **状态监控**：控制器复制数据过程中，主机可通过状态字段实时监控传输进度
- **早期访问**：无需初始化队列或启用控制器即可访问，适合系统启动早期阶段

## 读取操作流程

启动分区的读取操作遵循一个三步流程，主机通过配置寄存器发起请求，控制器负责数据传输：

### 操作步骤

```text
第 1 步：配置目标缓冲区
   BPMBL.BMBBA ← 页对齐的主机内存地址（Boot Partition Memory Buffer Base Address）

第 2 步：选择读取范围
   BPRSEL ← { 分区 ID (BPID), 偏移量（4 KiB 单位）, 大小（4 KiB 单位）}
             |
             v
第 3 步：监控传输状态
   BPINFO.BRS（Boot Read Status）状态变化：
   
   00b → 无活动请求
   01b → 数据传输进行中
   10b → 传输成功完成
   11b → 传输出错（目标缓冲区内容未定义）
```

### 关键寄存器说明

| 寄存器字段 | 全称 | 功能说明 |
|-----------|------|---------|
| **BPMBL.BMBBA** | Boot Partition Memory Buffer Base Address | 指定主机端接收数据的内存地址，必须页对齐（4 KiB） |
| **BPRSEL** | Boot Partition Read Select | 选择读取的分区、起始偏移和数据大小 |
| **BPINFO.BRS** | Boot Read Status | 当前读取操作的状态码 |
| **BPINFO.ABPID** | Active Boot Partition ID | 当前活动分区的标识符（0 或 1） |
| **BPINFO.BPSZ** | Boot Partition Size | 每个分区的大小，以 128 KiB 为单位 |

**注意**：`BPINFO.ABPID` 字段报告当前活动的分区编号，`BPINFO.BPSZ` 字段以 128 KiB 为单位报告每个分区的大小，两个分区始终保持相同大小。

## 地址对齐与边界检查

启动分区的读取操作对地址对齐和访问范围有严格要求，以确保数据传输的正确性和安全性。

### 地址布局

```text
启动分区内存布局
+------------------------------------------------------------------+
| 分区起始 |<--- 偏移 × 4 KiB --->|<--- 大小 × 4 KiB --->|  分区结束 |
+------------------------------------------------------------------+
          ↑                       ↑                     ↑
          0                    读取起始              读取结束
```

### 对齐与边界要求

| 要求项 | 规则 | 说明 |
|-------|------|------|
| **偏移量粒度** | 4 KiB 对齐 | 偏移量参数以 4 KiB 为单位 |
| **大小粒度** | 4 KiB 对齐 | 大小参数以 4 KiB 为单位 |
| **目标地址对齐** | 页对齐（4 KiB） | BPMBL 地址的低 12 位必须为 0（保留位） |
| **边界检查** | 偏移 + 大小 ≤ 分区大小 | 读取范围不得超出分区边界 |

### 越界处理

如果请求的读取范围超出分区边界：
- 控制器**不会传输任何数据**
- `BPINFO.BRS` 状态字段将报告错误（11b）
- 目标缓冲区内容处于未定义状态

## 日志页读取方式（只读路径）

除了通过 `BPRSEL` 寄存器直接读取，NVMe 还提供了一种基于 Get Log Page 命令的只读访问路径，适合需要标准化接口的场景。

### Get Log Page 接口

```text
发起命令
   Get Log Page, LID=15h, 指定 BPID（分区 ID）
              ↓
返回数据格式
+---------------------- 16 字节头部 ----------------------+--------------------+
| LID | BPINFO（活动分区 ID + 分区大小，128 KiB 单位） | 完整分区数据       |
+--------------------------------------------------------+--------------------+
|<------------- 固定头部信息 --------------->|<--- BPSZ × 128 KiB 数据 --->|
```

### 接口特性

| 特性 | 说明 |
|------|------|
| **命令类型** | Get Log Page（标准日志页读取命令） |
| **日志 ID** | LID = 15h（十六进制） |
| **分区选择** | 通过 `CDW10.BPID` 字段指定读取分区 0 或 1 |
| **返回内容** | 16 字节头部 + 完整分区数据（BPSZ × 128 KiB） |
| **头部信息** | 包含日志 ID、当前活动分区 ID、分区大小 |
| **副作用** | 不会修改 `BPINFO`、`BPRSEL` 或 `BPMBL` 寄存器状态 |

### 两种读取方式对比

| 对比项 | 寄存器方式（BPRSEL） | 日志页方式（Get Log Page） |
|-------|---------------------|--------------------------|
| **灵活性** | 可指定任意偏移和大小 | 只能读取完整分区 |
| **早期访问** | 支持启动早期访问 | 需要队列初始化 |
| **标准化** | 控制器特定寄存器 | 标准 NVMe 命令 |
| **状态影响** | 会更新 BPINFO.BRS | 不影响任何寄存器状态 |

**注意**：日志页布局已与规范图 267-268 核对确认。

## 写保护机制

启动分区提供了写保护机制，用于防止启动代码被恶意或意外修改。每个分区可以独立配置保护状态。

### 状态转换图

```text
系统上电/重新上电
      ↓
[Write Locked - 写锁定]  ←---Set Features 命令-→  [Write Unlocked - 写解锁]
      ↑                                                    |
      |                                                    |
      |                    Set Features 命令               |
      +------------- [Locked Until Power Cycle] ←---------+
                          永久锁定直到下次上电
                                   |
                          下次上电周期后
                                   ↓
                          [Write Locked - 写锁定]
```

### 状态说明

| 状态 | 英文名称 | 说明 | 持久性 |
|------|---------|------|--------|
| **写锁定** | Write Locked | 默认状态，禁止写入操作 | 重新上电后保持 |
| **写解锁** | Write Unlocked | 允许写入操作 | 重启或复位后恢复为写锁定 |
| **电源周期锁定** | Locked Until Power Cycle | 强制锁定直到下次完整断电 | 可穿越控制器级别复位 |
| **RPMB 控制** | RPMB Controlled | 由 RPMB 机制控制（Feature 无法覆盖） | 根据 RPMB 机制规则 |

### 重要特性

- **默认状态**：所有分区在上电后默认为写锁定状态
- **独立控制**：两个启动分区可以独立配置不同的保护状态
- **状态持久性**：电源周期锁定和 RPMB 控制状态会拒绝通过 Feature 命令修改
- **复位行为**：电源周期锁定状态在下次完整上电后恢复为写锁定

## Feature ID 85h：写保护配置

NVMe 定义了专门的 Feature ID 用于控制启动分区的写保护状态。

### Feature 功能说明

| 属性 | 值 |
|------|---|
| **Feature ID** | 85h（十六进制） |
| **名称** | Boot Partition Write Protection Config |
| **作用范围** | 每个分区独立配置 |
| **命令类型** | Set Features / Get Features |

### 可配置的保护动作

| 动作代码 | 英文名称 | 中文说明 | 效果 |
|---------|---------|---------|------|
| **000b** | No Change | 保持当前状态 | 不修改分区的写保护状态（仅用于 Set，Get 不返回） |
| **001b** | Write Unlock | 写解锁 | 允许写入操作（直到下次复位） |
| **010b** | Write Lock | 写锁定 | 禁止写入操作 |
| **011b** | Lock Until Power Cycle | 电源周期锁定 | 强制锁定直到下次完整断电 |
| **100b** | RPMB Controlled | RPMB 控制 | 仅用于 Get，表示由 RPMB 机制控制 |

### 特殊行为说明

#### 默认状态
- 分区在每次上电周期之后默认恢复为 **Write Locked（写锁定）** 状态
- 这提供了一个安全的默认配置

#### RPMB 控制的可见性
- 当 RPMB 机制处于活动状态时：
  - Get Features 会返回 `100b` 表示 RPMB 控制状态
  - Set Features 无法修改状态（请求会被拒绝）
  - 无法通过此 Feature 直接启用或禁用 RPMB

#### 电源周期锁定的不可变性
- 一旦设置为 **Locked Until Power Cycle** 状态
- 在下一次完整上电周期之前，该状态无法通过 Feature 命令更改
- 这提供了最高级别的临时保护

#### Set Features 特殊值
- `000b` (No Change) 仅用于 Set Features 命令，表示不修改当前状态
- Get Features 永远不会返回 `000b` 值

## 总结

启动分区是 NVMe 规范中为系统启动场景设计的专用存储区域，具有以下核心价值：

### 设计优势

1. **双分区冗余**：提供主备分区，确保固件更新失败时仍有可用的启动代码
2. **早期可访问**：无需完整初始化即可访问，满足 BIOS/固件的早期启动需求
3. **灵活保护**：提供多级写保护机制，适应不同安全需求
4. **灵活读取**：支持寄存器直接访问和标准日志页两种读取方式

### 典型应用场景

- **系统固件存储**：存储 UEFI、BIOS 或平台初始化代码
- **安全启动**：配合写保护机制实现可信启动链
- **固件备份与恢复**：利用双分区机制实现安全的固件更新和回滚
- **平台配置数据**：存储不易变更的平台级配置信息

### 实现建议

1. **读取实现**：优先使用寄存器方式以获得最佳性能和灵活性
2. **写保护策略**：根据安全需求选择合适的保护级别，默认使用写锁定状态
3. **错误处理**：实现完善的边界检查和状态验证，处理传输失败场景
4. **双分区管理**：充分利用双分区设计，实现安全的更新和回滚机制
