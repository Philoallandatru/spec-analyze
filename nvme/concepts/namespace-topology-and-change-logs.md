# 命名空间拓扑与变更日志

## 概述

在 NVMe 规范中，有两类日志用于追踪命名空间（Namespace）的组织结构和变化：

- **命名空间拓扑日志**（Namespace Topology Logs）：描述哪些子系统（Subsystems）、可达性组（Reachability Groups）以及可达性关联（Reachability Associations）参与了命名空间的访问
- **已分配命名空间变更列表**（Changed Allocated Namespace List）：报告控制器（Controller）可见的命名空间清单发生的变更

这些机制帮助主机（Host）理解命名空间的分布情况，以及如何在不同命名空间之间执行操作。

[PDF pp. 288-293](../_source/pages/page-288.md)

## 概念模型

为了更好地理解这些日志的结构，我们可以用以下层次关系来表达：

### 分散式命名空间视图

```text
分散式命名空间（Dispersed Namespace，指定 NSID）
└── 参与的 NVM 子系统 NQN 列表
```

当一个命名空间被标记为"分散式"时，它可能由多个 NVM 子系统共同提供服务。每个参与的子系统通过其唯一的 NQN（NVMe Qualified Name）来标识。

### 控制器可见拓扑

```text
控制器视角下的拓扑结构
└── 可达性关联（Reachability Association）
    ├── 定义：共享相同操作特征的一组可达性组
    └── 包含的可达性组（Reachability Group）
        ├── 定义：共享相同可达性属性的命名空间集合
        └── 成员：连接到该组的命名空间 NSID 列表
```

这个层次结构说明了：
- **可达性组**：将具有相同可达性属性的命名空间归为一组
- **可达性关联**：将多个可达性组关联起来，表示它们之间的操作关系（如是否支持快速复制）

### 已分配命名空间变更记录

```text
已分配命名空间的变更
├── 正常情况：最多记录 1024 个发生变更的 NSID
└── 溢出情况：当变更超过 1024 项时
    ├── 第一项为溢出哨兵值 FFFFFFFFh
    └── 其余项填充为零
```

> **说明**：以上层次结构概括了规范中的 Figures 270 与 274-278 的内容，详见 PDF 第 288-293 页。

[PDF pp. 288-293](../_source/pages/page-288.md)

## 日志规则与约束

### 日志类型对比

| 日志类型 | 成员与排序规则 | 一致性保证 |
|---------|--------------|----------|
| **分散式参与者列表**<br>（Log Page `17h`） | • 每个参与者占用 256 字节存储其 NQN<br>• 第 0 项是处理该命令的控制器所属子系统 | 当参与者被添加或移除时，生成计数器（Generation Counter）会递增 |
| **可达性组日志**<br>（Log Page `1Ah`） | • 仅包含连接到处理该命令的控制器的命名空间所属的组<br>• NSID 按升序排列 | 分段读取后需重新读取头部，如果全局变更计数不同则需重试 |
| **可达性关联日志**<br>（Log Page `1Bh`） | • 仅包含满足条件的可达性组所参与的关联<br>• RGID 按升序排列 | 使用与可达性组日志相同的头部校验协议 |
| **已分配命名空间变更列表**<br>（Log Page `1Ch`） | • 最多记录 1024 个发生变更的 NSID | 当变更数超过 1024 项时，第一项设为 `FFFFFFFFh`，其余项填零 |

[PDF pp. 288-293](../_source/pages/page-288.md)

### 分散式参与者日志的限制

访问分散式参与者日志时，有以下限制：

- **不接受的 NSID 值**：
  - `0h`（表示所有命名空间）
  - `FFFFFFFFh`（保留值）
- **命名空间要求**：目标命名空间必须被标记为"分散式"（Dispersed）类型

每个返回的 NQN 与对应参与者的 `SUBNQN` 字段相匹配。至于主机如何发现这些远程参与者，则不在 NVMe 规范的范围内（通常通过 NVMe-oF Discovery 服务实现）。

[PDF pp. 288-289](../_source/pages/page-288.md)

### 描述符优化标志

为了减少数据传输量，规范定义了两个优化标志：

| 标志 | 含义 | 效果 |
|-----|------|-----|
| `RGO=1` | Reachability Group Omit | 在可达性组描述符中省略 NSID 列表 |
| `RAO=1` | Reachability Association Omit | 在可达性关联描述符中省略 RGID（Reachability Group ID）列表 |

当支持索引偏移（Index Offset）时：
- 索引 0 对应日志头部（Log Header）
- 索引 1、2、3... 依次对应后续的描述符

[PDF pp. 290-292](../_source/pages/page-290.md)

### 变更计数器的行为

规范定义了两类变更计数器：

#### 全局变更计数

- **初始值**：控制器级别复位（Controller Level Reset）后从零开始
- **回绕行为**：递增到最大值后回绕至零

#### 描述符局部计数（可选）

- **初始值**：从一开始
- **回绕行为**：递增到最大值后回绕至一
- **特殊值零的含义**：控制器不支持该局部计数功能，主机需要通过内容比对来检测变化

[PDF pp. 290-293](../_source/pages/page-290.md)

### 可达性关联特征值

关联特征（Association Characteristic）字段定义了可达性组之间的操作能力：

| 特征值 | 含义 | 说明 |
|-------|------|-----|
| `01h` | 基本可达性 | 表示命名空间之间可以相互访问，但不保证性能特征 |
| `02h` | 不支持快速复制 | 成员组所代表的命名空间之间**不支持**快速复制（Fast Copy）操作 |
| `03h` | 支持快速复制 | 成员组所代表的命名空间之间**支持**快速复制操作 |

[PDF p. 293](../_source/pages/page-293.md)

### 已分配命名空间变更列表的范围

该列表涵盖自上次读取以来发生的所有影响命名空间的变更事件：

| 变更事件类型 | 说明 |
|------------|------|
| Identify Namespace 数据变化 | 命名空间的属性发生改变 |
| 命名空间连接（Attach） | 命名空间被连接到控制器 |
| 命名空间断开（Detach） | 命名空间从控制器断开 |
| 命名空间创建（Create） | 新建命名空间 |
| 命名空间删除（Delete） | 删除命名空间 |
