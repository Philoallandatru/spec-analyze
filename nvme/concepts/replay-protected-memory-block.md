# 重放保护内存块（Replay Protected Memory Block, RPMB）

重放保护内存块（RPMB）是控制器可访问的非命名空间存储区域，它通过为每个目标（target）配置独立的密钥、随机数回显（nonce echo）、单调递增的写计数器以及消息认证码（MAC）来保护已认证的读写操作，防止篡改和重放攻击。

[规范 PDF 第 569-573 页](../_source/pages/page-569.md)

## 安全模型

RPMB 的安全机制基于以下多层保护：

```text
安全预置阶段
    为每个 RPMB 目标编程一个不可读的密钥
              |
              v
主机请求阶段
    目标 + 随机数 + 计数器 + 地址 + 扇区数 + 数据
              |
              v
安全验证
    ├─ HMAC 验证请求真实性
    ├─ 随机数绑定读响应（防止重放）
    └─ 计数器绑定每次成功写入（防止回滚）
```

### 密钥管理

**密钥编程规则：**
- 密钥编程应在**安全预置环境**中完成（因为密钥编程本身未经过认证）
- 密钥一次性写入，写入后不可读取、不可修改
- 每个目标有独立的本地密钥
- 在密钥编程完成之前，任何访问都会返回"认证密钥尚未编程"错误

### 写计数器机制

**防回滚保护：**
- 每次成功的已认证数据写入都会递增一个 32 位计数器
- 计数器不可重置，单调递增
- 计数器饱和值为 `FFFFFFFFh`，达到后无法继续写入

[规范 PDF 第 569、572-573 页](../_source/pages/page-569.md)

## 目标与传输机制

### RPMB 目标特性

RPMB 目标（target）具有以下特征：
- **独立于命名空间**：不属于任何命名空间
- **可共享**：可以由子系统中的多个控制器共享
- **独立密钥**：每个目标需要自己的密钥

### 命令传输方式

RPMB 操作通过安全协议命令进行：
- **Security Send**：携带请求到 RPMB 目标
- **Security Receive**：取回响应
- 两者都使用 `NSID=0`（不针对特定命名空间）
- 帧中的 `NSSF` 与目标必须一致

[规范 PDF 第 570、573-574 页](../_source/pages/page-570.md)

### 请求响应流程

```text
Security Send(请求) ─────> 控制器/RPMB 目标
       |
       └──[可选：结果读取 Security Send]
               |
               v
Security Receive(响应/结果) <──── 控制器/RPMB 目标
```

**关键特性：**
- 对不同目标的请求可以交错执行
- 需要保序的主机会等待每个 Security 命令完成
- 较大的请求或响应可以跨越多个 Security 命令
- Security 命令必须等到其 RPMB 消息操作完成后才能成功完成

[规范 PDF 第 573-574 页](../_source/pages/page-570.md)

## 目标持久内容

每个 RPMB 目标包含两类持久内容：

| 内容类型 | 大小 | 持久性与访问特性 |
|---------|------|------------------|
| **认证密钥** | 由认证方法决定<br>（如 HMAC-SHA256 为 256 位） | • 一次性写入<br>• 不可擦除或读取<br>• 目标本地存储 |
| **数据区域** | 128 KiB 到 32 MiB<br>（由 Identify 报告） | • 已认证可读/可写<br>• 不可擦除<br>• 通过 HMAC 保护 |

## RPMB 帧格式与结果约定

### 帧结构

```text
RPMB 帧（256 字节前缀）
┌──────┬──────┬────────┬───────┬─────────┬─────────┬──────────┬────────┬──────┐
│ 填充 │ MAC/ │ 目标ID │随机数 │ 计数器  │  地址   │ 扇区数   │ 结果   │ 类型 │
│      │ 密钥 │        │       │         │         │          │        │      │
└──────┴──────┴────────┴───────┴─────────┴─────────┴──────────┴────────┴──────┘
                                                              + 可选数据（M 字节）

HMAC 输入范围：目标..类型（字节 223..255）|| 数据
               （排除填充字节与 MAC 字段）
```

### HMAC-SHA256 认证

对于 HMAC-SHA256 认证方法：
- 所选目标持有 256 位密钥
- MAC 按字节顺序覆盖上述帧字段
- 验证失败会导致操作被拒绝

### 操作结果分类

操作结果字段包含以下错误分类：
- 认证失败
- 计数器错误
- 地址错误
- 写入失败
- 读取失败
- 密钥缺失
- 配置无效

**特殊状态位：**独立的状态位报告写计数器是否已过期（达到饱和值）。

[规范 PDF 第 572-574 页](../_source/pages/page-569.md)

## 设备配置块（Device Configuration Block）

### 目标 0 的特殊用途

目标 0 包含一个特殊的 512 字节设备配置块（DCB），用于控制：

| 控制项 | 功能 |
|-------|------|
| **RPMB 引导分区保护** | 永久启用后，控制每个引导分区的锁定状态 |
| **命名空间写保护** | 控制进入"断电前写保护"或"永久写保护"状态 |

**重要特性：**
- 写保护控制位在上电循环或控制器级别重置时清零
- 无效的保护转换返回 RPMB 写入失败，而非静默改变状态
- DCB 拥有独立的写计数器，不与数据区计数器共享

[规范 PDF 第 570-572 页](../_source/pages/page-570.md)

## 密钥编程序列

密钥编程是一个三命令序列：

```text
步骤 1：Security Send 0001h（密钥）
              ↓
步骤 2：Security Send 0005h（结果请求）
              ↓
步骤 3：Security Receive 0100h（编程结果）
```

**关键点：**
- 密钥编程按目标进行
- 密钥一旦编程，无法通过 RPMB 撤销
- 必须在安全环境中完成

[规范 PDF 第 574-575 页](../_source/pages/page-574.md)

## 已认证数据操作

### 读取计数器

```text
Security Send 0002h(随机数)
        ↓
Security Receive 0200h(随机数, 计数器, MAC)
```

### 写入数据

```text
步骤 1：Security Send 0003h(计数器, 地址, 数据, MAC)
              ↓
步骤 2：Security Send 0005h（结果请求）
              ↓
步骤 3：Security Receive 0300h(结果, 新计数器, MAC)
```

**写入验证流程：**
1. 控制器检查计数器是否过期
2. 验证地址有效性
3. 验证 MAC
4. 核对计数器是否等于当前计数器
5. 失败时数据不变，成功时数据修改且计数器递增 1

**重要提示：**
- Security Send 完成仅表明 RPMB 处理已完成
- 不代表编程成功
- 主机必须取回并校验独立的已签名结果

[规范 PDF 第 575-577 页](../_source/pages/page-575.md)

### 读取数据

```text
Security Send 0004h(随机数, 地址, 扇区数)
        ↓
Security Receive 0400h(随机数, 数据, 结果, MAC)
```

**读取保护机制：**
- 响应绑定到主机的随机数（防止重放）
- 返回地址、扇区数、数据、结果和控制器 MAC
- **验证返回的 MAC 是主机的责任**，不是控制器接口的责任

[规范 PDF 第 575-578 页](../_source/pages/page-575.md)

## 已认证设备配置块操作

### DCB 写入（仅目标 0）

```text
步骤 1：Security Send 0006h(计数器, 512 字节 DCB, MAC)
              ↓
步骤 2：Security Send 0005h（结果请求）
              ↓
步骤 3：Security Receive 0600h(结果, 新 DCB 计数器, MAC)
```

**DCB 写入特性：**
- DCB 拥有独立的不可重置写计数器
- 独立于 RPMB 目标 0 的数据区计数器
- DCB 写入不会使数据区计数器递增
- 写入会校验计数器过期和 MAC
- **不能禁用已启用的引导分区保护**

### DCB 读取（仅目标 0）

```text
Security Send 0007h(随机数)
        ↓
Security Receive 0700h(随机数, DCB, DCB 计数器, MAC)
```

**DCB 读写约束：**
- 固定在目标 0
- 固定为一个 512 字节数据块
- 与数据写一样，Security Send 成功完成不保证 DCB 写成功
- 必须取回已签名的结果进行验证
